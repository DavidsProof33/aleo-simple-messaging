# Contributing Guidelines

Thank you for your interest in contributing to the `aleo-simple-messaging` project.  
This repository is intentionally minimal and educational, and the contribution rules reflect that simplicity.

---

## 1. Reporting Issues

If you encounter a bug:

1. Open an **Issue** on GitHub.  
2. Include the following information:
   - the file or component where the issue occurs,
   - clear steps to reproduce the problem,
   - your environment details (WSL/Windows, Leo version, Rust version, etc.).

---

## 2. Suggesting Improvements

We welcome suggestions such as:

- new examples,  
- code cleanups or simplifications,  
- documentation enhancements,  
- minor feature additions.

You may open an Issue or submit a Pull Request directly.

---

## 3. Pull Requests

Please follow these simple guidelines:

1. **Keep PRs small and focused.**  
2. Modify only what the PR title describes.  
3. Update documentation (`README` or `docs/`) if your change affects behavior.  
4. Ensure Rust code builds successfully (`cargo build`).  
5. Ensure the Leo program remains buildable (`leo build`).  

---

## 4. Code Style

While this project is intentionally simple, we encourage:

- clarity over complexity,  
- short, readable functions,  
- comments where necessary,  
- keeping the educational purpose in mind.
